## Firstly, What is Linux From Scratch?

Linux from scratch (often called 'LFS') is a project that aims to teach the user how to make a complete GNU/Linux operating system from the ground up.
Unlike many Linux distrobutions, LFS does not give you any pre-built packages, you have to build every package and all of its dependencies yourself.
The end result is a fully Linux system that contains only the software that you intentionally installed, every library, utility, compiler and configuration exists because you built and configured it.

Most modern linux distributions come with many packages precompiled and configured by the maintainers, while this is good for general compatibility with a wide aray of host machine configurations, it also hides a lot of what the system is actually doing and how it is working, however, lnux from scratch does not have this level of abstraction.

By manually compiling every component, you gain an understanding of many of the core fundimentals of linux, such as:
  * How compilers and toolchains are built
  * How the kernel interacts with the system
  * How libraries depend on each other
  * The role of shells, core utils, and system software
  * Why the software must be built in a specific order

## What can one learn from LFS

Working through LFS develops knowledge in several areas, including:
  * The GNU build system (configure, make, and make install)
  * Cross-compilation and temporary toolchains
  * Shared and static libraries
  * Dynamic linking
  * Filesystem hierarchy standards
  * Kernel compilation and configuration
  * Package dependencies
  * System initialization

Although the finished operating system is minimal compared to modern distributions, the concepts learned apply directly to virtually every Linux distribution in existence.
